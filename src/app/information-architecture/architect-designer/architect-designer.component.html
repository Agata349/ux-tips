What is the difference between UX Designer and UX Architect?

A UX Architect (Information Architect) is focused on a high-level view of the product and is responsible for creating a structure and flow based on user and market research. UX Architects can conduct research on their own or with the help of the research team if needed.

A UX Designer is responsible for designing user interfaces and makes a product usable. In many companies UX Designers are responsible for early research and creating user personas. Usually they also conduct necessary usability testing.

UX Architect vs. UX Designer - main differences

UX Architect UX Designer
looks at the bigger picture focuses on the details
decides how to structure the content, creates wireframes creates high-fidelity prototype with the proper interactions
focuses on navigation and user flow creates the user interfaces and interactions for each view
reviews research - considers what features and content the user needs reviews research - wants to know how the user interacts with the elements
answers the question: Who are the users, and what do they need? answers the question: Who are the users, and how do we meet their need?

Typical workflow

  1. UX Architect analyzes market and user research in order to establish how to structure the content.
  2. UX Architect puts together a blueprint (wireframes & prototypes) for the UX Designer.
  3. UX Designer analyzes user research and the blueprint and starts to design each user interface.
  4. During usability studies - UX Architect focuses more on accessibility (how the user navigate through the page and how accesses content). UX Designer wants to see how the user interacts with content.
  5. Once a product is live - UX Architects will double check if the content is up-to-date. They will focus on accessibility and recommend updates. According to this recommendation, UX Designer will analyze interactions and fix each screen to make it more user friendly.
