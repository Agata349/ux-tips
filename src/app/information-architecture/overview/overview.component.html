What is Information Architecture?

Information Architecture (IA) means consciously organizing the content and flow of a website, app or system. You can observe it in everyday life. A very simple example - names of the aisles in the grocery store. That organization should me meaningful, which means it's based on what you know about the people using this product. In other words - what people expectations are.

It's based on:

  • How content will be organized
  • What content will be called (labeling)
  • What and how much they expect to find
  • How they expect to move though it

Therefore we can distinguish four core components of IA:

  • Organization Schemes and Structures: How information is categorized and structured.
  • Labeling Systems: How information is represented.
  • Navigation Systems: How users browse or move trough information.
  • Search Systems: How users look for information.

Strategic content

The content should deliver a specific message to end users, make them feel a certain way. Should be:

  • Relevant - Are the users going to identify with it?
  • Appropriate - Is the style appropriate? Is the message consistent?
  • Useful - Is it valuable and meaningful to the people consuming it?

Identifying content requirements

  • Who is going to create content?
  • Who is going to edit and approve it?
  • Who is going to manage and update it?
  • What are competitors doing?
  • Will people think this is useful and valuable?
  • How will they find all of this content?
  • How many different kinds are there?
  • Where does it fit in our established IA?
  • How will we display it?
  • Are there form factor restrictions (e.g. mobile device)?

5 states affect content with examples

  1. Physical - Which device do the users use? Are they performing other tasks at the same time?
  2. Environmental - Are they viewing content on their phone as they are walking down the street?
  3. Preferential - Are there any learning or physical disabilities that should be considered?
  4. Emotional - What's motivating their activity?
  5. Cognitive - Is it suitable for all education or skill levels?

Content classification methods

  1. Chronological - appropriate for time based content
  2. Alphabetical - great when users need to find something fast, e.g. indexes
  3. Geographical - use when content is specifically related to a physical place
  4. Most popular - great for interest-based browsing
  5. Process order - use when content represent a process

Information Architecture must be user-centered

DON'Ts

  • Don't use acronyms (when you need to use them, at least expand them)
  • Don't use difficult terminology and jargon
  • Don't overlap categories
  • Don't make up words and terms
  • Don't make the user think too much!

DO's

  • Think and predict! If I were a user, what would I call it?
  • The easier labels, the better
  • Names should be descriptive
  • Think about your colleagues when naming the files - keep it consistent with the project

Primary and secondary navigation

Primary navigation - content that most users are interested in. For example categories like "Men", "Women", "Kids" on the clothing store website.
Secondary navigation - content that doesn't serve the primary goal. For instance tabs like "Home", "Contact Us".

Global and local navigation

Global navigation - appears on every page of the website. It allows users to switch between top-level categories easily, from any location.
Local navigation - only appears on category pages and is specific to that particular category. It helps users preview lower-level content.
